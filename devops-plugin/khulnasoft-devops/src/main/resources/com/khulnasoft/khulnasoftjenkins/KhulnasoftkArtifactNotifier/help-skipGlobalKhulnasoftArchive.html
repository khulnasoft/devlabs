When enabled, this job will skip the global Khulnasoft archiving behavior configured at the system level. This is useful when you want this specific job to handle artifact archiving differently or not at all.

Use cases:

  • Jobs that should not send artifacts to Khulnasoft
  • Jobs that handle artifact archiving through custom scripts or other means
  • Jobs in development or testing phases where Khulnasoft integration is not needed

Note: This setting only affects the automatic archiving behavior. You can still manually trigger artifact archiving using the RunDelegate.archive() method in your build scripts.
